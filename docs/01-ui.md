## Implementation Plan — Sacred Discipline UI (Pass 1)

### What we are building

A mobile-first, dark-only Sacred Discipline UI on the existing Next.js static + Capacitor scaffold: design tokens, shared app shell, the three Stitch screens (Readiness Dashboard, Physical Readiness, Muhasaba Focus), and the missing screens (Sign-in, Intellect + narration detail, fitness questionnaire, workout session checklist, weekly Muhasaba audit) in the same visual language. Interaction is real (navigation, toggles, sheets, forms); **data is mock / in-memory only** — no SQLite, Preferences persistence, Supabase, pedometer, notifications, or LLM APIs in this pass.

### Language we agreed on

- **01-ui.md:** Phased implementation plan for this UI pass (scope, decisions, steps, done-when) — not a replacement for `ui-plan.md`
- **Build the UI:** Shell + Stitch screens + missing screens that look and navigate correctly
- **Mock data:** Hardcoded / in-memory demo values for rings, charts, scores, modules, prompts
- **Interactive:** Real UI state and routing; no disk persistence or network calls yet

### Decisions made

- **Pass scope:** Tokens + shell + three Stitch screens + full missing screens (same design language, mobile-first). Defer Societal Readiness as product feature; hide PDF download; no native widget; no real sync/AI backends
- **Missing screens:** Build for real (not stubs): Sign-in, Intellect feed + detail, fitness questionnaire, workout checklist/session, weekly Muhasaba report
- **Routing:** App Router `(tabs)` layout (`/`, `/physical/`, `/intellect/`, `/spiritual/`) + stack routes for subflows; tab bar on primary tabs only
- **Prayer log UI:** Five binary rows (Fajr → Isha), micro-journal sheet on complete (in-memory)
- **AGENTS.md:** Agents must read `docs/` before building

### Assumptions

- Settings screen is out of this pass (account affordance on top bar → Sign-in only)
- Motion: include the three core Framer Motion beats from `ui-plan.md` (READY pulse, ring draw-in, sync spin)
- Atmospheric / training images: local placeholders or solid tonal blocks if assets are unavailable; no remote Googleusercontent URLs in app code
- Spiritual footer copy: “Stored locally on device” (honest; encryption claim deferred)
- Lucide icons; install `framer-motion` + `lucide-react`
- Tailwind v4 CSS variables for Sacred Discipline tokens in `globals.css`

### How to build it

1. **Docs gate** — Update `AGENTS.md` so agents read `docs/` (especially `project-overview.md`, `ui-plan.md`, active `01-*.md`) before coding.
2. **Dependencies** — Add `lucide-react`, `framer-motion`.
3. **Tokens & fonts** — Map Sacred Discipline colors/spacing/radius into CSS variables; load Source Serif 4, Geist, JetBrains Mono in root layout; force `dark` class; base background `#0b1326`.
4. **Route architecture**
   ```
   src/app/
     (tabs)/layout.tsx          → AppTopBar + BottomTabs + safe-area padding
     (tabs)/page.tsx            → Readiness Dashboard
     (tabs)/physical/page.tsx
     (tabs)/intellect/page.tsx
     (tabs)/spiritual/page.tsx
     sign-in/page.tsx
     physical/questionnaire/page.tsx
     physical/session/page.tsx
     intellect/[id]/page.tsx
     spiritual/audit/page.tsx
   ```
5. **Shared components** (from `ui-plan.md` §5) — `AppTopBar`, `BottomTabs`, `ReadyPulse`, `ReadinessRing`, `CommandmentCard`, `PillarTile` / `PillarBanner`, `LogActionRow`, `MetricCard`, `StatusChip`, `DisciplineCard`, `PresenceRow`, `PromptComposer`, `TrainingModuleCard`, `WeeklyLoadChart`, `PrimaryButton`, `DisciplineButton`, `FAB`, plus sheets for journal + FAB actions.
6. **Mock module** — `src/lib/mock/` with readiness %, pillars, commandment, weekly bars, training modules, five prayers, prompt, audit blocks, Ghaybah narrations.
7. **Screen: Dashboard** — Ring, Daily Commandment, pillars (navigate to tabs), log action rows, atmospheric block, FAB action sheet.
8. **Screen: Physical** — Header, Readiness Load chart, Record Session → `/physical/session/`, module cards → session, Stillness strip; entry to questionnaire when “first run” mock flag is set.
9. **Screen: Spiritual** — Header + Hijri mock date, five `PresenceRow`s + journal sheet, Ready Summary (circle + INDEX; no PDF button), PromptComposer, micro metrics; link to `/spiritual/audit/`.
10. **Screen: Intellect** — Mobile-first Ghaybah feed using Commandment card DNA; detail route for full narration.
11. **Screen: Sign-in** — Brand wordmark, email + password fields, gold primary CTA, **no sign-up**; back to app.
12. **Screen: Questionnaire** — Multi-step mobile form (height, weight, baseline, medical, equipment); mock submit → Physical.
13. **Screen: Session checklist** — Interactive checklist for a mock workout plan; checks in memory only.
14. **Screen: Weekly audit** — Diagnostic / Remedy / Prescription sections; `CACHED LOCALLY` chip; optional skeleton.
15. **Motion** — READY pulse, ring draw-in, sync icon spin (sync is UI-only feedback).
16. **Capacitor check** — `npm run build` (static export) + `npx cap sync` to confirm shell still packs.

### Out of scope (this pass)

- `@capacitor/preferences`, SQLite, pedometer, local notifications
- Supabase auth/sync, OpenAI routes
- Real prayer-time engine / Hijri calendar library (mock strings OK)
- Societal Readiness as a product feature
- PDF export, native home-screen widget
- Light theme

### Done when

- [x] Four tabs navigate; shell matches Sacred Discipline tokens
- [x] Dashboard / Physical / Spiritual match Stitch layout hierarchy and copy structure
- [x] Sign-in, Intellect (+ detail), questionnaire, session checklist, weekly audit are usable mobile screens in the same language
- [x] Prayer toggles + journal + prompt edits work in memory
- [x] Static export build succeeds; no server-only Next features
- [x] `AGENTS.md` requires reading docs before build work
